# Panduan Kontribusi

Kami menggunakan spesifikasi [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/) untuk pesan commit.

## Format Commit
`<type>[optional scope]: <description>`

contoh: 
- `feat(api): menambahkan endpoint balita`
- `fix(ui): memperbaiki tata letak dashboard`
- `docs: memperbarui README`

## Tipe Commit
- `feat`: Fitur baru
- `fix`: Perbaikan bug
- `docs`: Dokumentasi saja
- `style`: Perubahan yang tidak memengaruhi logika (spasi, format, dsb.)
- `refactor`: Refactoring kode
- `perf`: Peningkatan performa
- `test`: Menambahkan atau memperbaiki test
- `chore`: Perubahan pada proses build, alat bantu, dll.
